.gitignore文件

.gitignore只能忽略那些原来没有被追踪的文件，如果需要新增忽略文件的话

误删本地分支恢复

删除分支的本质，其实是删除了.git/refs/heads/目录下的一个与分支同名文件，文件内容是一个分支所指向commit对象的sha-1值。也就是说真正删除的并不是commit对象本身，更为形象的说就是揭掉贴在commit对象上的标签，当然我们可以再贴上一个标签。
